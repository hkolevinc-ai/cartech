# CarTechBG catalog scraper

Scraper за `https://cartechbg.com/`, направен за GitHub Actions автоматизация.

## Какво извлича

Excel файлът е структуриран по изискания ред:

1. `Уникален SKU номер`
2. `Име на продукт`
3. `Категория`
4. `Описание от страницата на продукта`
5. `Изображение 1`
6. `Изображение 2`
7. `Изображение 3`
8. `Изображение 4`
9. `Изображение 5`
10. `Изображение 6`
11. `Изображение 7`
12. `Цена в евро`
13. `Линк към продукта`

## Важно за изображенията

Scraper-ът взима изображения само от WooCommerce product gallery:

- `.woocommerce-product-gallery`
- `.woocommerce-product-gallery__image`
- `data-large_image`, `data-src`, `data-thumb`, `src`, `href`

Не взима изображения от таба/блока с описание на продукта.

## Как работи

1. Обхожда категориите от менюто/снимките и запомня категорията за всеки продукт.
2. Проверява sitemap-и, за да не изпусне продуктови URL-и:
   - `/sitemap.xml`
   - `/sitemap_index.xml`
   - `/product-sitemap.xml`
   - `/product-sitemap2.xml`, `/product-sitemap3.xml`, ...
   - `/wp-sitemap-posts-product-1.xml`
3. За всеки продукт извлича SKU, име, категория, описание, изображения от product gallery, цена в евро и линк.
4. При вариативни продукти прави отделен ред за всяка вариация, когато има отделен variation SKU.
5. Записва резултата в `cartechbg_all_products.xlsx`.

## Как се пуска в GitHub

Качи/замени тези файлове в repository-то:

- `scraper.py`
- `requirements.txt`
- `.github/workflows/scrape.yml`

После отвори:

`Actions` → `Scrape CarTechBG catalog` → `Run workflow`

След приключване Excel файлът ще е в `Artifacts` като `cartechbg_all_products`.

## Настройки

Могат да се променят от workflow файла или като environment variables:

- `BASE_URL` — по подразбиране `https://cartechbg.com/`
- `OUTPUT_FILE` — по подразбиране `cartechbg_all_products.xlsx`
- `REQUEST_SLEEP` — пауза между заявките, по подразбиране `0.35` секунди
- `MAX_PRODUCTS` — лимит за тест, `0` означава без лимит
- `MAX_CATEGORY_PAGES` — максимум страници при обхождане на категории
- `MAX_IMAGES` — брой колони за изображения, по подразбиране `7`

## Локален тест

```bash
pip install -r requirements.txt
MAX_PRODUCTS=5 python scraper.py
```
